## 1. Request Summary

Please implement the Japanese Rice first runner chart dataset vertical slice.

The current Japanese Rice GUI wrapper already shows a weekly table:


## 8. Required Row Fields

Each row must include original fields:

```text
week
requested
capacity
accepted
blocked
```

Each row must include derived fields:

```text
shortage
unused_capacity
capacity_usage_ratio
blocked_ratio
capacity_usage_pct
blocked_pct
```

Definitions:

```text
shortage = blocked
unused_capacity = max(capacity - accepted, 0)
capacity_usage_ratio = accepted / capacity if capacity > 0 else 0
blocked_ratio = blocked / requested if requested > 0 else 0
capacity_usage_pct = capacity_usage_ratio * 100
blocked_pct = blocked_ratio * 100
```

All numeric values should remain numeric, not formatted strings.

```python

    "weekly_rows": [

    ],
    "totals": {"requested": 10, "capacity": 0, "accepted": 0, "blocked": 10},

```

```text

blocked_ratio = 1.0

blocked_pct = 100
```

Expected totals:

```text
capacity_usage_ratio = 0

capacity_usage_pct = 0

```

No division by zero should occur.
